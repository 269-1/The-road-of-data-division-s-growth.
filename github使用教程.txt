1.网络连接问题:
使用下面网站:https://tool.chinaz.com/dns/ ->输入框中输入github.com,不断点击查看其给出的ip地址和TTL时间(时间越短越好)
多次测试得到一个最小时间对应的IP地址后,打开/c/windows/system32/drivers/etc/host
再最后添加我们得到的IP地址+ github.com
保存并关闭
2.创建个人账户
3.创建个人仓库(repositories)
4.安装git软件
第一次提交文件到github仓库时（仓库中无文件）
(1)添加远程仓库
打开git命令行(git bash) 并输入
ssh-keygen -t rsa -C "你创建的账号如example@qq.com"
再三次回车得到密钥,成功的话会在 ~/ 下生成 .ssh 文件夹
在windows找到~/.ssh下的id_rsa.pub打开 id_rsa.pub，复制里面的 key。
进入github setting 中找到ssh and GPG keys然后添加我们复制的密钥并添加.再回到git命令行输入
ssh -T git@github.com 查看是否已经连接成功.(出现successful即可)



(2)设置提交代码时的用户信息
git config --global user.name "example"
git config --global user.email "example@qq.com"


(3)创建本地仓库数据
git mkdir bigdata#先创建本地目录
cd bigdata#进入到本地目录
#如果此时想上传文件
   1.pwd得到文件路径,找到下面的bigdata文件夹
   2.将要上传的文件放到bigdata文件夹下,也可不在bigdata中放置,但是需要对应文件的绝对路径并且git add加入绝对路径
git init #本地仓库初始化
git add .# .是添加该目录下的所有文件,也可指定具体的某个文件
git commit -m  "说明文字"




(4)连接本地仓库和远程仓库
在github 仓库code栏找到类似如下链接:git@github.com:269-1/The-road-of-data-division-s-growth..git将其复制
git remote add origin +复制的内容(实际是远程仓库的链接)
(git remote -v可以查看已有的远程库信息,删除可以使用git remote rm origin)



(5)提交代码到远程仓库(github)
git push -u origin master#推送本地仓库数据到远程仓库



克隆git clone 代码仓库网页链接(如:git://github.com/schacon/grit.git)


本地目录/工作台->远程仓库需要:
1.add工作台->暂存池
2.commit 暂存池->本地仓库(需要初始化 git init())
3.push 本地仓库->远程仓库
pull实际是拉回到缓冲池中。


非第一次使用github仓库（仓库中有文件）
删除远程仓库中的内容：
可以借助本地仓库与远程仓库的同步机制实现，即先将远程仓库中的内容拉回到本地之后，通过使用git rm -r --cached +文件名删除缓冲池中的文件再使用commit提交到本地仓库更新本地仓库，之后再使用push推送到远程仓库。就可以实现对远程仓库文件的删除。
在有文件的条件下，向远程仓库添加文件：
1.将github上的文件拉回到本地目录和缓冲区中git pull --rebase origin master2.使用git add ...向缓冲池中加入需要新添加的文件3.提交文件git commit -m "本次操作描述"到本地仓库4.推送文件到远程仓库github上git push -u origin master。
当缓冲区有改变时需要提交这次改变并且push到远程仓库之后才能使用pull 。相当于缓冲区，本地仓库，远程仓库是紧密相连的。
